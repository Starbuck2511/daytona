Leistungen:

Das Leistungsangebot erstreckt sich von der Analyse, Konzeption und Implementierung bis zur Betreuung und Weiterentwicklung der Produktivsysteme. Projektmanagement und Koordination externer Services gehören ebenfalls zum Leistungsumfang.

Details

Development

Programmierung Frontend

Clientseitige Programmierung auf Basis von JavaScript. Vorzugsweise ES6 und höher. Gerne auch mit TypeScript, welches ein Superset von JavaScript darstellt und beim Entwickeln zusätzlichen Komfort und Sicherheit bietet.

Neben der Programmierung für UI Komponenten auf Websites kommt JavaScript auch für die Realisierung von webbasierten Apps oder Single Page Applications (SPA) zum Einsatz. Verwendete Frameworks: Angular 2 (und höher) und Ionic.

Programmierung Backend

Zum Einsatz kommt in der Regel PHP7, welches der serverseitigen Script-Sprache, viele neue Features bei der Entwicklung bereitstellt. Zudem ist mit PHP7 prinzipiell eine bessere Performance erreichbar.

Die Applikations-Entwicklung findet mit dem objektorientierten Ansatz statt. Zum Einsatz kommen Best Practices wie die Verwendung von Design Patterns und automatisierten Tests (Unit Tests). Basis für die meisten Anwendungen bildet das Symfony-Framework.

Realisierung UX/UI

Erstellung von Benutzeroberflächen und -schnittstellen basierend auf aktuellen Webstandards wie HTML5 und CSS3. Vorzugsweise Applikationen mit Responsive Design unter Berücksichtigung des Mobile First Ansatzes, welcher neben Design und Performance auch die Informations-Architektur berücksichtigt.

Des Weiteren kommen Präprozessoren wie SASS, LESS, Stylus zum Einsatz für die optimale Strukturierung des eingesetzten CSS.

Toolchain

Für Frontend- und Backendprojekte werden Build-Tools wie webpack und gulp eingesetzt. Diese basieren auf node.js und werden mit Hilfe von npm installiert. Prinzipiell wird jegliches Dependency-Management mit den dafür vorgesehenen Tools wie npm oder Composer abgebildet.

Git und git-flow

Alle Projekte werden mit Git verwaltet um jederzeit Zugriff auf die unterschiedlichen Code-Versionen eines Projektes zu haben. Das ermöglicht maximale Transparenz in der Entwicklung für alle Mitglieder des Teams.

Zudem bietet es einfache Möglichkeiten verschiedene neue Features oder Patches zur parallel aktuellen Code-Basis zu entwickeln. Als Prozess-Tool wird git-flow verwendet.

DevOps

VMs und Continuous Integration

Für die Entwicklung kommen meist virtuelle Maschinen (VM) zum Einsatz welche mit Vagrant oder Docker aufgesetzt werden. Dies bietet der Vorteil, die Systemumgebung des Live-Systems für alle Mitglieder des Teams nahezu identisch bereit stellen zu können.

Zudem werden nach Möglichkeit automatisierte Tests des neu erstellten Codes und automatische Deployments auf Testsysteme durchgeführt, um eine möglichst hohe Qualität sicher zustellen (z.B. mit Jenkins).

Datenbanken

SQL und NoSQL

Entwurf und Implementierung von relationalen (z.B. MySQL) und nicht relationalen, sogenannten NoSQL Datenbankanwendungen (z.B. MongoDB).
